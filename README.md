### ūü§Ė AI Chatbot ‚ÄĒ Finance-Only Assistant (Groq API)

The standout feature of TrackFin. A floating, glassmorphic AI assistant powered by **LLaMA 3.3 70B via Groq**.

```
User: "how to make maggie"
Bot:  "I'm your Finance Assistant. I can only help with financial
       topics like budgets, expenses, income, savings, and goals."

User: "how much did I spend this month?"
Bot:  "You've spent ‚āĻ12,400 this month across Food (‚āĻ4,200),
       Travel (‚āĻ3,100), and Utilities (‚āĻ5,100)."
```

**What makes it production-grade:**
- ‚úÖ **Profanity filter** ‚ÄĒ blocks abusive input before it reaches the API
- ‚úÖ **Topic guard** ‚ÄĒ only responds to finance-related queries
- ‚úÖ **User-scoped** ‚ÄĒ reads only the logged-in user's MongoDB data via JWT
- ‚úÖ **Conversation memory** ‚ÄĒ full history sent on every request
- ‚úÖ **Strong system prompt** ‚ÄĒ AI cannot be jailbroken into off-topic answers
- ‚úÖ **Natural language transactions** ‚ÄĒ *"Spent ‚āĻ500 on coffee today"* ‚Üí auto-extracted as expense

### ūüĒź Authentication & Security

| Feature | Implementation |
|---------|---------------|
| Login / Signup | JWT Access + Refresh Token pattern |
| Token Storage | Refresh token in `HttpOnly` cookie (XSS-safe) |
| Password Reset | OTP via Nodemailer (Gmail App Password) |
| Route Protection | `authMiddleware` on all private API routes |
| Password Hashing | BCrypt with salt rounds |
| API Key Safety | Groq key stays server-side, never exposed to frontend |

## ‚öôÔłŹ Local Setup

### Prerequisites
- Node.js v18+
- MongoDB Atlas account (free tier works)
- Groq API key ‚ÄĒ get one free at [console.groq.com](https://console.groq.com)
- Gmail account with App Password enabled

<br/>

### Step 1 ‚ÄĒ Clone the repo

```bash
git clone https://github.com/your-username/finance-tracker.git
cd finance-tracker
```

### Step 2 ‚ÄĒ Backend environment

Create `backend/.env`:

```env
# Server
PORT=5000
NODE_ENV=development

# Database
MONGO_URI=mongodb+srv://<user>:<password>@cluster.mongodb.net/trackfin

# JWT Secrets (use any long random strings)
JWT_SECRET=your_super_secret_access_key_here
JWT_REFRESH_SECRET=your_super_secret_refresh_key_here

# Email (Gmail App Password ‚ÄĒ 16 chars, no spaces)
EMAIL_USER=your.email@gmail.com
EMAIL_PASS=abcd efgh ijkl mnop

# AI
GROQ_API_KEY=gsk_your_groq_api_key_here
```

### Step 3 ‚ÄĒ Frontend environment

Create `frontend/.env.local`:

```env
NEXT_PUBLIC_API_URL=http://localhost:5000
```

### Step 4 ‚ÄĒ Install & run

Open **two terminals**:

```bash
# Terminal 1 ‚ÄĒ Backend
cd backend
npm install
npm run dev
# Runs on http://localhost:5000
```

```bash
# Terminal 2 ‚ÄĒ Frontend
cd frontend
npm install
npm run dev
# Runs on http://localhost:3000
```

Visit `http://localhost:3000` ‚ÄĒ you're live. ūüöÄ

## ūüĒí Security Practices

- API keys live only in `.env` ‚ÄĒ never in frontend code
- JWT refresh tokens stored in `HttpOnly` cookies ‚ÄĒ not `localStorage`
- All `/api/ai/*` and `/api/transactions/*` routes require valid JWT
- Groq API called **server-side only** ‚ÄĒ the browser never touches it
- Profanity + topic filters run **before** the LLM ‚ÄĒ no wasted API calls
- Passwords hashed with BCrypt before storage ‚ÄĒ never stored in plain text
