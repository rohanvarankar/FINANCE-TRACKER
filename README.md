## вңЁ Feature Highlights

<br/>

### рҹӨ– AI Chatbot вҖ” Finance-Only Assistant (Groq API)

The standout feature of TrackFin. A floating, glassmorphic AI assistant powered by **LLaMA 3.3 70B via Groq**.

```
User: "how to make maggie"
Bot:  "I'm your Finance Assistant. I can only help with financial
       topics like budgets, expenses, income, savings, and goals."

User: "how much did I spend this month?"
Bot:  "You've spent вӮ№12,400 this month across Food (вӮ№4,200),
       Travel (вӮ№3,100), and Utilities (вӮ№5,100)."
```

**What makes it production-grade:**
- вң… **Profanity filter** вҖ” blocks abusive input before it reaches the API
- вң… **Topic guard** вҖ” only responds to finance-related queries
- вң… **User-scoped** вҖ” reads only the logged-in user's MongoDB data via JWT
- вң… **Conversation memory** вҖ” full history sent on every request
- вң… **Strong system prompt** вҖ” AI cannot be jailbroken into off-topic answers
- вң… **Natural language transactions** вҖ” *"Spent вӮ№500 on coffee today"* вҶ’ auto-extracted as expense

<br/>

### рҹ”җ Authentication & Security

| Feature | Implementation |
|---------|---------------|
| Login / Signup | JWT Access + Refresh Token pattern |
| Token Storage | Refresh token in `HttpOnly` cookie (XSS-safe) |
| Password Reset | OTP via Nodemailer (Gmail App Password) |
| Route Protection | `authMiddleware` on all private API routes |
| Password Hashing | BCrypt with salt rounds |
| API Key Safety | Groq key stays server-side, never exposed to frontend |

## вҡҷпёҸ Local Setup

### Prerequisites
- Node.js v18+
- MongoDB Atlas account (free tier works)
- Groq API key вҖ” get one free at [console.groq.com](https://console.groq.com)
- Gmail account with App Password enabled

<br/>

### Step 1 вҖ” Clone the repo

```bash
git clone https://github.com/your-username/finance-tracker.git
cd finance-tracker
```

### Step 2 вҖ” Backend environment

Create `backend/.env`:

```env
# Server
PORT=5000
NODE_ENV=development

# Database
MONGO_URI=mongodb+srv://<user>:<password>@cluster.mongodb.net/trackfin

# JWT Secrets (use any long random strings)
JWT_SECRET=your_super_secret_access_key_here
JWT_REFRESH_SECRET=your_super_secret_refresh_key_here

# Email (Gmail App Password вҖ” 16 chars, no spaces)
EMAIL_USER=your.email@gmail.com
EMAIL_PASS=abcd efgh ijkl mnop

# AI
GROQ_API_KEY=gsk_your_groq_api_key_here
```

### Step 3 вҖ” Frontend environment

Create `frontend/.env.local`:

```env
NEXT_PUBLIC_API_URL=http://localhost:5000
```

### Step 4 вҖ” Install & run

Open **two terminals**:

```bash
# Terminal 1 вҖ” Backend
cd backend
npm install
npm run dev
# Runs on http://localhost:5000
```

```bash
# Terminal 2 вҖ” Frontend
cd frontend
npm install
npm run dev
# Runs on http://localhost:3000
```

Visit `http://localhost:3000` вҖ” you're live. рҹҡҖ

## рҹ§  How the AI Chatbot Works (For Technical Readers)

The chatbot uses a **two-layer filtering system** before any Groq API call is made:

```
User message
     в”Ӯ
     в–ј
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ  Layer 1: Profanity     в”Ӯв”Җв”Җв”Җв”Җ blocked в”Җв”ҖвҶ’  "Please keep it respectful."
в”Ӯ  (regex word boundary)  в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”¬в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
             в”Ӯ clean
             в–ј
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ  Layer 2: Topic Guard   в”Ӯв”Җв”Җв”Җв”Җ off-topic в”ҖвҶ’ "I only help with finance."
в”Ӯ  (finance keyword list) в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”¬в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
             в”Ӯ finance вң“
             в–ј
  Fetch user's MongoDB data
  (transactions + summary вҖ” scoped to JWT userId)
             в”Ӯ
             в–ј
  Build messages array:
  [ system prompt, ...history, new message ]
             в”Ӯ
             в–ј
     Groq API вҶ’ LLaMA 3.3 70B
             в”Ӯ
             в–ј
     Reply вҶ’ Frontend
```

**The system prompt** enforces rules at the model level вҖ” even if a user crafts a clever message past the keyword filter, the LLM is instructed to refuse non-finance answers and never fabricate financial data.

**OpenAI-compatible format** вҖ” Groq uses the same `/v1/chat/completions` endpoint format as OpenAI, making it trivial to swap providers if needed.

## рҹ”‘ Key API Endpoints

```
POST   /api/auth/register          Register new user
POST   /api/auth/login             Login + receive JWT
POST   /api/auth/verify-otp        Verify email OTP
POST   /api/auth/forgot-password   Trigger OTP for password reset

GET    /api/transactions           Get user's transactions (paginated)
POST   /api/transactions           Add new transaction
PUT    /api/transactions/:id       Edit transaction
DELETE /api/transactions/:id       Delete transaction

GET    /api/budgets                Get all budgets
POST   /api/goals                  Create a savings goal

POST   /api/ai/chatbot             Finance chatbot (auth required)
GET    /api/ai/advisor             Monthly AI tips (auth required)
POST   /api/ai/process-text        Natural language вҶ’ transaction JSON
```

## рҹ”’ Security Practices

- API keys live only in `.env` вҖ” never in frontend code
- JWT refresh tokens stored in `HttpOnly` cookies вҖ” not `localStorage`
- All `/api/ai/*` and `/api/transactions/*` routes require valid JWT
- Groq API called **server-side only** вҖ” the browser never touches it
- Profanity + topic filters run **before** the LLM вҖ” no wasted API calls
- Passwords hashed with BCrypt before storage вҖ” never stored in plain text
